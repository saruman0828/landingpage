# サイト計測と文言改善の運用

## 計測する流れ

1. `page_view`：ページを開いた
2. `case_view`：個別事例を読んだ
3. `cta_click`：無料診断・相談ボタンを押した
4. `form_start`：フォーム入力を始めた
5. `lead_completed`：送信後の完了ページへ到達した

補助指標として、`scroll_depth` の50・90、`case_open`、`outbound_click`、`form_submit_error`も見る。

## 毎週見る数字

| 指標 | 計算 | 分かること |
| --- | --- | --- |
| 事例閲覧率 | case_view ÷ page_view | 実績への関心 |
| CTAクリック率 | cta_click ÷ page_view | 提案とボタンの強さ |
| フォーム開始率 | form_start ÷ page_view | 相談意欲 |
| フォーム完了率 | lead_completed ÷ form_start | 入力負担やエラー |
| 問い合わせ率 | lead_completed ÷ page_view | サイト全体の成果 |
| 90%読了率 | scroll_depth（90）÷ page_view | 長い説明が読まれた割合 |

## 文言改善の手順

1. 直近14〜28日を集計する。
2. 最も離脱が大きい1か所を選ぶ。
3. 一度に変えるのは、見出し、説明、CTAのいずれか1つだけにする。
4. `data-copy-version` を新しい版名へ変える。
5. 同程度の訪問数が集まるまで待つ。
6. `copy_view` と `lead_completed` を版別に比較する。
7. 問い合わせ率が高く、送信失敗率が悪化していない版を残す。

## 数字の読み方

- 50%読了率が低い：冒頭が長い、誰向けか分かりにくい。
- 事例閲覧率が低い：事例への導線や見出しが弱い。
- CTAクリック率が低い：相談で得られるもの、料金、安心材料が不足している。
- フォーム開始率は高いが完了率が低い：項目が多い、必須条件が重い、送信エラーがある。
- 事例閲覧後の問い合わせ率が高い：トップページから近い事例へ誘導する価値が高い。

## 最初の基準版

- トップページ：`main-clear-v1`
- AI研修ページ：`training-clear-v1`

次回は `main-clear-v2` のように番号を上げる。日付ではなく版名で比較すると、集計期間がずれても追いやすい。

## Vercel管理画面側の準備

1. 対象プロジェクトの Analytics を開く。
2. Web Analytics を有効にする。
3. 変更後のデプロイを本番へ反映する。
4. 本番ページを開き、ページ表示が記録されるかを見る。
5. Pro以上の場合は Events で `cta_click`、`form_start`、`lead_completed` を見る。

カスタムイベントを利用できないプランでは、ページ表示と参照元はVercelで見られる。行動経路とファネルを詳しく見る場合はUmamiまたはPlausibleを追加する。
