# Web表現の参考と今回の反省

作成日: 2026-06-20

## 前回までの問題

「そうそう、それ」と言わせようとして、表現を日常会話まで落としすぎた。

その結果、想定クライアントである中小企業の社長・後継者・役員から見ると、以下のように見える危険があった。

- 経営課題ではなく、作業改善の小ネタに見える
- 社長の責任や判断の重さを軽く扱っているように見える
- AI導入支援というより、一般的な業務効率化メモに見える
- 価格帯に見合う専門性が伝わらない

## 参考にするWeb表現の方向

### Basecamp

参考点:

- 読者の状況を言い当てるが、幼い言い方はしない。
- 「仕事が散らばっている」「決定、会話、ファイル、期限が分散している」という構造的な問題として語る。

反映:

- 社長の確認を単なる雑務ではなく、「判断基準が共有されていない経営課題」として表現する。

出典:

- https://basecamp.com/

### SmartHR / カオナビ

参考点:

- 機能説明の前に、組織課題、業務効率化、戦略人事、データ活用など、経営寄りの言葉で価値を伝える。
- 課題別・業界別の入口を作り、読者が自社に引き寄せやすい構成にしている。

反映:

- 業種別あるあるではなく、権限委譲、品質担保、教育負担、対応品質、月次確認という経営課題に言い換える。

出典:

- https://smarthr.jp/
- https://www.kaonavi.jp/

### 7shifts

参考点:

- レストラン向けでも、単なる現場あるあるではなく、採用、教育、シフト、給与、定着を一連の運営課題として見せている。
- 「チーム運営の基盤」という見せ方で、単発作業ではなく経営管理に近づけている。

反映:

- 飲食・店舗の表現は、仕込みやシフトの小話ではなく、売上予測、原価、人員、廃棄、残業、口コミ対応が利益に影響する問題として扱う。

出典:

- https://www.7shifts.com/

## 今回の表現方針

- 読者を「AI初心者」扱いしない。
- 社長の時間削減だけでなく、権限委譲と品質担保を主題にする。
- 業務改善ではなく、判断基準の移譲として語る。
- AI研修批判ではなく、研修だけでは運用設計が残らないという問題にする。
- 納品物は「資料」ではなく、会社に残る運用資産として見せる。
