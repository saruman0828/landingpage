# サブページ企画

作成日: 2026-06-20

## 結論

新規客向けには、トップページだけでなく「悩み別サブページ」を作るべき。主ターゲットは中小企業の社長だが、社長以外の責任者にも刺さるように、ページタイトルは「社長依存」を軸にしつつ本文では「社長・責任者」と広げる。

理由:

- 初見客は自分に関係あるページだけ読む。
- 「AI導入支援」より「社員からの確認を減らす」の方が検索・広告・紹介で刺さりやすい。
- 料金や問い合わせ前の不安は、トップページだけでは解消しきれない。

## 最優先ページ1: 社長依存チェックページ

想定ファイル:

- `diagnosis.html`

目的:

新規客に「これは自社の問題だ」と気づかせ、無料診断へつなげる。

ファーストビュー:

「社員が何度も聞くのは、確認場所がないからです。」

本文:

「社長が忙しい会社ほど、仕事量そのものより『社長や責任者に聞かないと進まない仕事』が増えています。まずは、どの仕事が経営者・責任者に戻ってきているか確認しましょう。」

チェック項目:

- 同じ質問に週3回以上答えている
- 社長しか返せないメールがある
- 新人教育が毎回口頭になっている
- 見積・請求の最後は社長確認になる
- 社員が迷った時に見る資料がない
- 問い合わせ返信の品質が人によって違う
- AIに興味はあるが、何から始めるか分からない
- 社長以外の責任者が、業務改善を任されている

診断結果:

- 0〜1個: まず1業務だけ整備
- 2〜3個: 社長確認を減らす余地あり
- 4個以上: 社長依存が成長のブレーキになっている可能性

CTA:

「30分で、社長が手放せる仕事を一緒に仕分ける」

## 最優先ページ2: 改善事例ページ

想定ファイル:

- `cases.html`

目的:

何をしてくれるのかを具体化し、問い合わせ前の不安を減らす。

ファーストビュー:

「AIで減らすのは、人ではなく、毎回社長や責任者に戻ってくる確認仕事です。」

事例の型:

- 業種
- よくある悩み
- 作るもの
- 導入後の状態
- 削減時間の目安

掲載事例:

### 不動産

悩み:

毎朝のメール確認、物件情報確認、問い合わせ返信が担当者ごとにバラバラ。

作るもの:

- 問い合わせ分類表
- 返信テンプレート
- 朝の確認チェックリスト
- 社長確認が必要な条件

見出し:

「毎朝の確認業務を、社員が整理して社長が見る流れへ。」

### 建設・工務店

悩み:

見積、現場確認、協力会社への連絡、工程確認が社長に集中。

作るもの:

- 現場ヒアリングシート
- 概算見積チェックリスト
- 協力会社への確認文
- 社長確認ライン

見出し:

「現場で聞くこと、社長に確認することを分ける。」

### 飲食店

悩み:

仕込み量、シフト、原価、口コミ返信が店長や社長の勘に依存。

作るもの:

- 仕込み判断表
- 曜日・天気・イベント確認表
- 口コミ返信テンプレート
- 売上振り返りフォーマット

見出し:

「勘だけに頼らず、判断材料を残す店舗運営へ。」

### 士業・事務

悩み:

顧客資料の確認、進捗連絡、面談メモ、よくある質問が毎回手作業。

作るもの:

- 顧客資料チェックリスト
- 面談メモ要約フォーマット
- 進捗連絡テンプレート
- FAQ

見出し:

「毎回ゼロから確認する仕事を、型にする。」

CTA:

「自社ならどの業務から始めるべきか相談する」

## 最優先ページ3: AI研修で失敗した人向けページ

想定ファイル:

- `training.html`

目的:

他の講義・講師との差別化。研修ではなく実装支援であることを伝える。

ファーストビュー:

「AI研修を受けても、現場で使われない理由があります。」

問題提起:

- ツール紹介だけで終わった
- プロンプト集をもらったが使っていない
- 社員に任せたら使う人と使わない人に分かれた
- セキュリティが不安で止まった
- 結局、社長だけがAIを使っている

解決策:

「必要なのは講義ではなく、御社の業務で使う成果物です。」

成果物:

- 実際のメール返信テンプレート
- 社内FAQ
- チェックリスト
- AI指示文
- 入力ルール
- 社員向けの使い方

比較表:

| よくあるAI研修 | 右腕AIの支援 |
| --- | --- |
| ツール紹介 | 御社の業務を整理 |
| プロンプト集 | 実際の返信文・チェックリスト |
| 受講者任せ | 社員が使う運用まで設計 |
| 便利さ中心 | 情報管理と確認手順も設計 |
| その日で終わり | 会社に資料が残る |

CTA:

「研修ではなく、まず1業務を使える形にする」

## ページ4: 料金・費用対効果ページ

想定ファイル:

- `roi.html`

目的:

価格を見た新規客の不安を減らす。

ファーストビュー:

「毎日1時間の社長確認は、年間250時間です。」

構成:

1. 社長時間の試算
   - 30分: 年125時間
   - 1時間: 年250時間
   - 2時間: 年500時間
2. 社員の迷う時間
   - 社員5名が毎日10分迷うと年約208時間
3. 作った資料は会社に残る
4. 料金プラン別に向いている会社
5. 無料診断で現実的な削減見込みを出す

CTA:

「自社の削減時間を30分で見積もる」

## ページ5: 無料診断ページ

想定ファイル:

- `consultation.html`

目的:

問い合わせへの心理的ハードルを下げる。

ファーストビュー:

「30分で、社長が手放せる仕事を3つに仕分けます。」

診断でやること:

1. 今、社長や責任者に戻っている仕事を聞く
2. AI・テンプレート化できる業務を分ける
3. 最初に整える1業務を決める
4. 必要な成果物を提案する
5. 無理に契約しなくてよい

フォーム:

- メールまたは電話番号
- 相談したいこと
- 任意メモ

CTA:

「無料診断を申し込む」
