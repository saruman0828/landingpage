# FAX受注入力の改善事例 出典台帳

## 対象記事

- 公開HTML: `case-studies/fax-order-intake.html`
- テーマ: 食品・資材卸で午前中に集中するFAX注文の入力
- 主人公: 食品・資材卸の受注担当
- 匿名化: 実名企業・製品名・所在地・固有の商品名は公開HTMLへ掲載しない

## 本質設計シート

- 読者: FAX受注が残る食品・包装資材などの卸売業の経営者、受注責任者
- 痛む時間帯: 始業後から午前中
- 止まる仕事: FAX注文の読み取り、商品コードへの読み替え、基幹システムへの転記、納期返信
- 隠れたストレス: 電話による中断、桁間違い、規格違い、納品先違い、返信遅れ
- 入力: FAX、商品コード表、取引先ごとの帳票
- AIの出力: 受注CSV、読取エラー一覧、納期返信案
- 担当者の承認点: 商品、数量、納品先、納期、基幹システム登録、取引先への送信
- 結果: 1件あたり数分の転記を減らし、月数十時間規模の改善余地を作る
- 時間の使い道: 欠品時の代替提案、倉庫との納期調整、電話対応
- 画像の役割: 注文内容と商品情報を人が照合し、受注処理を進める業務イメージ

## 出典1

- URL: https://www.smartocr.jp/cases/case022.html
- ページ名: 月1000件のFAX注文書を自動入力。毎月約50時間の作業が3分の1に短縮できた
- 提供元: 株式会社インフォディオ「スマートOCR」導入事例
- 参照日: 2026-07-12
- 公表事例の業種: 業務用洗剤・洗浄剤、固形燃料などの製造販売
- 公表事例の規模: 月5万〜6万件の受注の約4分の1がFAX。OCR処理は月約1,000件
- 公表された課題:
  - 取引先ごとに注文書の書式が異なる
  - 取引先の注文コードを自社コードへ変換する必要がある
  - FAX注文を基幹システムへ手入力していた
- 公表された運用:
  - FAXを専用フォルダへ保存
  - OCRが読み取り、担当者が読取結果を修正して確定
  - CSVを出力し、基幹システムへ取り込む
  - 読み取りは100%ではないため、人が目を通す
- ベンダーが示す効果:
  - 月約50時間の作業が平均で約3分の1
  - 手入力から読取結果のチェックと修正へ移行
  - 熟練度による処理速度の差と心理的負担が減少
- 記事への反映: CSV出力、商品コード表との照合、読取エラーを担当者が修正する流れ

## 出典2

- URL: https://www.infomart.co.jp/ai-ocr/case/17.asp
- ページ名: FAX発注書の自動受注で入力・納期返信の作業時間を短縮
- 提供元: 株式会社インフォマート「発注書AI-OCR」導入事例
- 参照日: 2026-07-12
- 公表事例の業種: 業務用食肉の製造販売
- 公表事例の規模: 月約2,000件の受注のうち、FAXが約1,200枚
- 公表された課題:
  - 取引先独自の商品名や旧商品名を自社商品へ読み替える
  - 規格違いの商品を判別する
  - 商品、発注番号などを基幹システムへ手入力する
  - 注文書ごとのレイアウト差や複数納品先を含む帳票は手作業が残る
- 公表された運用:
  - AI-OCRがFAX注文をデータ化し、CSVを基幹システムへ取り込む
  - 納品日を入れた返信をテンプレートから作成する
  - システムが読み取った内容を人が見てから送信する
- ベンダーが示す効果:
  - 発注書の入力から納期返信まで、1件約4分から約1分
  - FAX注文の8割を自動受注処理
  - 入力中心から読取結果を見る運用へ移行
- 記事への反映: 取引先独自の商品名、規格違い、納期返信案、特殊帳票を読取エラー一覧へ残す考え方

## 公開記事での数値の扱い

- `月50時間が約3分の1`、`1件4分から1分`は、各ベンダーが特定企業への取材で示した数値であり、本記事の匿名ケースが達成した実績とは書かない。
- 公開HTMLでは、`月数十時間規模`、`1件数分の処理が約1分になった例`という規模感へ抽象化する。
- 効果はFAX件数、帳票の種類、手書きやかすれ、商品マスター、基幹システム連携で変わる旨を併記する。
- 導入時は、一部取引先で1件の処理時間、読取エラーの修正時間、納期返信までの時間を計測する。

## 安全表現と公開ルール

- AI-OCRが商品、数量、納品先、納期を確定したとは書かない。
- 受注担当が原票を読み、商品、数量、納品先、納期を承認する。
- 基幹システムへの登録と取引先への返信は、担当者の承認後に行う。
- 公開HTMLには出典URL、実名企業、製品名、固有の商品名を掲載しない。
- 画像説明は業務上の役割を示し、画像内の人物、場所、端末、帳票の細部を断定しない。
