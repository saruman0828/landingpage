# さらに広域調査 海外・国内追加事例

作成日: 2026-07-07

## 見方

前回までの資料では、建設、町工場、店舗、営業、顧客対応を中心に整理した。今回はさらに広げて、医療、金融、法務、保険、物流、製造、教育、店舗運営、開発、行政・大学事務まで見た。

重要なのは業種名ではなく、「人が何に時間を取られていたか」。

- 長い文書を読む
- 必要情報を探す
- 会話や現場情報を記録する
- 複数システムから転記する
- 下書きを作る
- ルート、在庫、検査などの候補を出す
- よくある問い合わせを一次対応する

この単位で見ると、海外大企業の事例も中小企業へ置き換えやすい。

## 今回追加で見えた「強い事例」の傾向

### 1. 医療のAI記録は、社長の右腕AIにかなり近い

医療は専門性が高いが、AIがやっていることは「会話を聞き、記録にし、人が確認する」。これは建築点検、営業商談、士業面談、保険相談、不動産内見にも置き換えられる。

使える言い換え:

- 診療メモ -> 商談メモ、点検メモ、相談記録
- 患者向け説明 -> 顧客向け説明、次回案内
- 時間外記録 -> 帰社後の報告書、夜のメール返信

### 2. 金融・法務は「探す・読む・論点化」が本質

Morgan Stanley、CoCounsel、JPMorgan、Bank of Americaは、大量の文書や顧客情報を素早く扱う例。中小企業では、契約書、過去見積、顧客履歴、補助金要領、社内規程に置き換えられる。

使える言い換え:

- 10万件の投資資料 -> 過去の提案書、見積、施工履歴
- 契約レビュー -> 取引条件、注意点、修正候補
- 顧客セルフサービス -> よくある質問、予約、支払い確認

### 3. 物流・製造は「人の動きと確認順序」を変えている

UPS、DHL、BMW、Toyotaは、AIが文章を書くというより、順番、検査箇所、在庫配置、知識の取り出しを改善している。これは現場型ビジネスに強い。

使える言い換え:

- ルート最適化 -> 訪問順、配送順、現場巡回順
- 品質検査 -> 点検写真の確認、施工不備チェック
- 熟練知識AI -> ベテランの手順、判断基準、過去トラブル

### 4. 教育・店舗は「聞けるマニュアル」が強い

Plex Coffee、Khanmigo、Preply、Toyota manualの共通点は、固定のPDFや研修を、質問できる形に変えていること。これは多店舗、現場、営業、バックオフィスに使いやすい。

使える言い換え:

- 店舗ハンドブック -> 接客ルール、発注手順、クレーム対応
- 教師アシスタント -> 社内教育、OJT補助
- 車のマニュアル -> 機械、設備、業務システムの操作案内

## 注意が必要な事例

- 医療、法務、金融、保険は高リスク領域。AIが判断したように書かない
- 顧客対応は、AIが完結するより「人へ引き継げる」設計を強調する
- 大企業の総削減額は中小企業に大きすぎるので、1人あたり、1件あたり、月あたりに翻訳する
- ベンダー事例は成功側に寄りやすいので、LPでは「公開事例では」と表現する
- AI導入の目的を「人を減らす」ではなく「人が判断・接客・営業に戻る」に置く
