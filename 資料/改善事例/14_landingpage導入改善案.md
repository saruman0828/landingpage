## 結論

最優先は事例数の追加ではない。現在の強みである「30件の業務別ケース」を、信頼できる根拠、探しやすさ、相談時の文脈引継ぎへつなぐ改善が先。

特に重要な順は次の5点。

1. 実案件・公開事例・想定シナリオの区別
2. 出典つき公開事例と測定条件の提示
3. 事例ページから相談完了までの計測と文脈引継ぎ
4. 30件を悩み別に探せる絞り込み
5. 壊れているトップナビゲーションの修正

## 現状で活かすべき強み

- トップページに自社支援実績5件、削減時間の目安、算出上の注意書きがある。
- 30件の業務別ケースがあり、建設、経理、採用、問い合わせ、店舗、医療周辺などを広くカバー。
- 各事例に Before / After、AIが先に進める範囲、人が担う範囲、導入ステップがある。
- 「まず1業務」「人が最後に確認」「30分無料診断」は参考資料の成功原則と一致。
- 問い合わせフォーム、LINE代替導線、トップと研修ページの基本イベント計測は導入済み。

これらは作り直さず、信用・検索性・計測を足す方が合理的。

## 公開事例枠に向く候補

参考資料の中では、中小企業が自社に置き換えやすい順に次が有力。

1. A-FACTORY: シフト作成・入荷管理、年間650時間削減
2. 鈴木鉄工所: 7名規模、RPAで月40時間削減
3. direct建設報告書DX: 報告書作成が半分以下、方式によって1/5
4. 大林組: AI配筋検査で作業時間約36%縮減
5. 板橋区: AI-OCRで年間約1,480時間削減見込み

掲載時は資料内の注意に従い、「公開発表によると」「同社発表」「削減見込み」など、発表主体と測定状態を明記。

## 推奨する実装順

### 第1段階: 信用と不具合

1. 事例種別の明示と「30社」表現の精査
2. ナビゲーション修正
3. 公開事例3件と出典追加
4. 自社実績の算出条件追加

### 第2段階: 問い合わせ導線

1. 事例別CTAとフォーム文脈引継ぎ
2. 全事例ページの分析イベント
3. 悩み別フィルター
4. AI/人の役割分担表

### 第3段階: 商談前の不安解消

1. 成果物見本
2. 1業務パイロットの範囲と測定方法
3. セキュリティ・FAQ
4. 料金判断材料
